relational db -> in form of tables
these tables are connected using relationships 
E.g;
tables: 
1. user table
2. posts table
3. comments table

relationships:
1 post -> 1 user
1 comment -> 1 post
1 user -> man posts

non relational db -> not arranged in form of tables
    -> uses documents as well as key-value pairs

why use relational db?
    -> data has clear structure
    -> relationships btn data  
    -> handles transactions, joins
    -> strong validation at db level

E.g;
-> in banking, e commerce, CRM  

non relational is used in data that changes very often, docs independent, not much joins used
